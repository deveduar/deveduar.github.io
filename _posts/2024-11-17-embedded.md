## Conceptos fundamentales
- Firmware 
	- Software de bajo nivel que controla directamente el hardware de un dispositivo.
	- Se almacena en memoria no volátil (flash).
	- Permite la comunicación entre sensores, actuadores y sistemas superiores.
	- Lenguajes y entornos comunes:
		- ccp  
		- [Rust](/desarrollo%20multiplataforma/rust/)
		- MicroPython
		- [Java ME](/desarrollo%20multiplataforma/java-me/)
- RTOS (FreeRTOS, Zephyr)
	- Sistemas operativos en tiempo real diseñados para microcontroladores.
	- Permiten ejecutar múltiples tareas con prioridades.
	- Uso típico en sistemas embebidos críticos (IoT, automoción, industria).
- Plataformas de desarrollo
	- Arduino
		- Fácil de usar, ideal para principiantes.
		- Gran comunidad y librerías.
	- ESP32
		- Microcontrolador potente con conectividad integrada.
	- Raspberry Pi Pico
		- Basado en RP2040.
		- Bajo costo y alto rendimiento para proyectos embebidos.

---

## Microcontroladores vs PC de placa única
- Microcontroladores
	- Diseñados para tareas específicas.
	- Bajo consumo energético.
	- Ejecutan firmware directamente.
- PC de placa única (SBC)
	- Ejecutan sistemas operativos completos (Linux).
	- Mayor capacidad de procesamiento.
	- Ejemplo destacado:
		- **Raspberry Pi**
			- CPU completa (no solo controlador).
			- Permite ejecutar aplicaciones complejas.
			- Usos:
				- Servidores locales (ej: Pi-hole).
				- Automatización del hogar.
				- Edge computing.

---

## Electrónica básica aplicada
- FPGA chip
	- Circuito integrado programable.
	- Permite diseñar hardware personalizado.
	- Usado en aplicaciones de alto rendimiento y prototipado.
	- Recurso:
		- [¿Qué es un FPGA? - YouTube](https://youtu.be/unERWj2lOvA) 
- Circuito integrado y puertas lógicas
	- Base de la electrónica digital.
	- Operaciones fundamentales:
		- AND, OR, NOT, NAND, NOR, XOR.
- Kit básico de electrónica
	- Resistencias
	- LEDs
	- Sensores básicos
	- Cables jumper
- Protoboard vs PCB
	- Protoboard
		- Prototipado sin soldadura.
		- Reutilizable.
	- PCB
		- Diseño permanente.
		- Más robusto y compacto.
- Microcontroladores con WiFi
	- Permiten conexión directa a internet.
	- Base de proyectos IoT.

---

## SoC (System on a Chip)
- Integración de CPU, memoria y periféricos en un solo chip.
- Ejemplo clave:
	- ESP32
		- Conectividad:
			- WiFi
			- Bluetooth
		- Alimentación:
			- 5V (entrada), lógica a 3.3V
		- Pines:
			- ~26 GPIO disponibles (dependiendo del modelo)
		- Arquitectura:
			- 32 bits
		- Comunicación:
			- 3 puertos seriales (UART)
		- Programación:
			- [cpp](/software%20engineering/cpp/) 
		- Interacción hardware:
			- Sensores y actuadores
				- Voltaje (3.3V)
				- Señal
				- Tierra (GND)
		- Aplicaciones:
			- Automatización
			- Domótica
			- Monitorización remota
			- Bots de notificaciones (ej: Telegram)

## Relaciones clave
- Firmware conecta hardware con software.
- IoT combina microcontroladores + conectividad.
- RTOS mejora la gestión de tareas en sistemas embebidos.
- SoC integra todo en un solo chip → mayor eficiencia.
- Seguridad es crítica en cualquier sistema conectado.

# IoT y sistemas embebidos — expansión avanzada

## Arquitectura de sistemas embebidos
- Capas de un sistema embebido
	- Hardware
		- Microcontrolador / SoC
		- Sensores y actuadores
	- Firmware
		- Control directo del hardware
		- Drivers (GPIO, I2C, SPI, UART)
	- Middleware
		- Comunicación entre módulos
		- Protocolos de red
	- Aplicación
		- Lógica del sistema
		- Automatización o interfaz
- Flujo típico
	- Sensor → procesamiento → decisión → actuador → comunicación

---

## Protocolos de comunicación (clave en IoT)
- Comunicación local (hardware)
	- I2C
		- Comunicación entre múltiples dispositivos usando 2 cables
	- SPI
		- Alta velocidad, más pines
	- UART
		- Comunicación serial simple (TX/RX)
- Comunicación en red
	- MQTT
		- Ligero, basado en publish/subscribe
		- Ideal para IoT
	- HTTP/HTTPS
		- Más pesado pero universal
	- WebSockets
		- Comunicación bidireccional en tiempo real
- Redes inalámbricas
	- WiFi
		- Alto consumo, alta velocidad
	- Bluetooth / BLE
		- Bajo consumo
	- LoRa
		- Largo alcance, baja velocidad
	- Zigbee
		- Redes malladas (mesh)

---

## Gestión de energía (crítico en IoT)
- Modos de consumo
	- Active mode
	- Sleep mode
	- Deep sleep (clave en ESP32)
- Técnicas de optimización
	- Reducir frecuencia de CPU
	- Apagar periféricos no usados
	- Uso de interrupciones en vez de polling
- Alimentación
	- Baterías (Li-ion, LiPo)
	- Reguladores de voltaje
	- Energía solar (proyectos autónomos)

---

## Sensores y actuadores (profundización)
- Sensores analógicos vs digitales
	- Analógicos
		- Requieren ADC
	- Digitales
		- Comunicación directa (I2C, SPI)
- Tipos comunes
	- Temperatura (DHT11, DHT22)
	- Movimiento (PIR)
	- Luz (LDR)
	- Presión, gas, humedad
- Actuadores
	- Relés
	- Motores (DC, servo, stepper)
	- LEDs
- Consideraciones
	- Ruido eléctrico
	- Calibración
	- Precisión vs costo

## Seguridad avanzada en IoT
- Superficie de ataque
	- Firmware vulnerable
	- Comunicaciones sin cifrar
	- Credenciales por defecto
- Técnicas de protección
	- TLS/SSL
	- Autenticación (tokens, certificados)
	- Secure boot
	- OTA segura (actualizaciones remotas)
- Hardening
	- Desactivar debug (UART/JTAG)
	- Protección de memoria
