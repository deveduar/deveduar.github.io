# Guia inicial devops
## Introducción a DevOps
DevOps es una **cultura**, **metodología** y **conjunto de prácticas** que integran desarrollo, operaciones y seguridad para ofrecer software de forma más **rápida, fiable y automatizada**. Su objetivo central es mejorar la **colaboración**, reducir **errores humanos**, aumentar la **calidad**, acelerar los **ciclos de entrega** y mejorar la **observabilidad** y el **rendimiento** de los sistemas.

## Principios Fundamentales
- **Automatización extrema**  
	*Automatización de builds, tests, despliegues, infraestructura y seguridad.*
- **Colaboración continua**  
	*Equipos multidisciplinares, transparencia y comunicación constante.*
- **Ciclo CI/CD**  
	*Flujos de integración y entrega frecuente con pipelines repetibles.*
- **Observabilidad integral**  
	*Logs, métricas, trazas distribuidas y visualización continua.*
- **Shift-left**  
	*Pruebas y seguridad introducidas desde etapas tempranas.*
- **Infraestructura como código (IaC)**  
	*Infraestructura declarativa y reproducible versionada en [git](/software%20engineering/git/).*

## Cadena de Valor DevOps (DevOps Toolchain)
Representa todos los procesos y herramientas que habilitan un ciclo de vida completo.

### Planificación
- Roadmaps, user stories, backlog.
- Integración con herramientas como Jira, GitLab Issues, GitHub Projects.

### Coding & Build
- Repositorios: [git](/software%20engineering/git/), [monorepo](/backend/monorepo/), [nx](/automatizacion%20y%20build/nx/).
- Automatización de compilación: [Automatizacion y Build](/automatizacion%20y%20build/automatizacion-y-build/).
- Dependencias, revisión de código, convenciones.

### Testeo
- Unitarias, integración, regresión, contrato, performance.
- Escaneos SAST, DAST, SCA → ver [DevSecOps](/devops/devsecops/).

### Integración Continua (CI)
- Pipelines con [CICD](/devops/cicd/), [gitlab](/cloud/gitlab/), [GitHub Actions](/devops/github-actions/), [bitbucket](/devops/bitbucket/).
- Validación automática tras cada push.
- Artefactos generados y versionados.

### Entrega y Despliegue (CD)
- Blue/Green, Canary, Rolling, Feature Flags.
- Orquestación en contenedores y VMs: [Virtualizacion](/devops/virtualizacion/), Kubernetes, vps.
- Despliegues en edge o serverless.

### Operación
- Gestión de logs, runtime y servidores: [Runtime Management](/backend/runtime-management/), apache tomcat, [netifly](/cloud/netifly/).
- Configuración de redes, balanceo y accesos: [Redes](/redes/redes/), [proxy](/backend/proxy/), vpn.

### Observabilidad
- Monitoreo, alertas, dashboards: [monitoreo](/monitoreo/monitoreo/).
- Trazabilidad y correlación con herramientas APM.
- Buenas prácticas: golden signals, SLIs, SLOs (ver [SRE Site Reliability Engineering](/devops/sre-site-reliability-engineering/)).

### Seguridad
- Hardening de sistemas.
- Auditoría, compliance, cifrado: [Seguridad Web y Certificados TLS](/devops/seguridad-web-y-certificados-tls/).
- Integración continua de controles: [DevSecOps](/devops/devsecops/).

## Arquitectura Base en un Entorno DevOps
- **Control de versiones**: repos y flujos GitOps.

- **Seguridad integrada**: políticas, escaneos y gestión de secretos.  
- **Entornos replicables**: staging, QA, producción, creación automática.  
- **Observabilidad unificada**: logs, métricas y trazas.

## Prácticas Avanzadas DevOps
### GitOps
- Despliegues declarativos basados en repositorios Git.
- Operadores como FluxCD o ArgoCD.

### Infraestructura Inmutable
- Cada despliegue crea nuevas imágenes/entornos en lugar de modificar los existentes.

### Service Mesh
- Control de tráfico entre microservicios.
- Mallas como Istio o Linkerd para retries, circuit breaking y observabilidad avanzada.

### Seguridad Zero Trust
- Ningún componente es confiable por defecto.
- Autenticación continua, políticas estrictas, segmentación.

### Multi-región y Alta Disponibilidad
- Replicación de estados.
- Estrategias activas/activas o activas/pasivas.
- DRP y planes de recuperación.

## Ejemplos de Pipelines DevOps Complejos
### Pipeline con Validaciones y Seguridad (CI/CD + DevSecOps)
1. Commit → Linter + Tests Unitarios  
2. Análisis SAST/SCA  
3. Build container + escaneo de imagen  
4. Tests de integración + e2e  
5. Despliegue Canary  
6. Validación automática mediante métricas SLO  
7. Promoción a producción

### Pipeline Multi-región con Service Mesh
1. Build y empaquetado  
2. Despliegue simultáneo en región primaria  
3. Inyección automática de sidecars  
4. Canary con reglas de tráfico del mesh  
5. Replicación en otra región  
6. Observación de latencia y errores por región  
7. Failover automático si es necesario

### Pipeline GitOps con Infraestructura como Código
1. Commit en repo IaC  
2. Validación de sintaxis y estática  
3. Plan de cambios  
4. Aprobación  
5. Aplicación automática por operador  
6. Auditoría de drift  
7. Alertas si difiere estado deseado vs real

# DevOps — Conceptos Avanzados y Expansiones

## Filosofías y Modelos Operativos
### Platform Engineering
- Enfoque moderno que construye plataformas autoservicio para desarrolladores.
- Reduce fricción operativa y estandariza pipelines, entornos y despliegues.
- Incluye plataformas internas (IDP — Internal Developer Platforms).

### NoOps
- Operaciones completamente automatizadas.
- Apoya modelos serverless, autoscaling y pipelines autónomos.
- Minimiza intervenciones manuales y reduce el coste operativo.

### GitOps Madurado
- Despliegues totalmente declarativos basados en estado deseado.
- Auditoría completa y rollbacks automáticos.
- Operadores inteligentes para sincronización continua (ArgoCD, FluxCD).

## Patrón de Integración y Entrega
### Progressive Delivery
- Evolución del CD tradicional.
- Combina feature flags + canary releases + monitoreo automatizado.
- Permite experimentación segura y validación de nuevas funcionalidades.

### Continuous Verification (CV)
- Validación automática basada en datos de producción.
- Usa métricas, logs, anomalías y errores en tiempo real para decidir si avanzar.
- Integración con modelos estadísticos y ML.

## Diseño de Arquitectura Asociada a DevOps
### Microfrontends
- Desacopla equipos de frontend para escalar DevOps al lado del cliente.
- Builds independientes, despliegues parciales y aislados.

### Event-Driven Architecture
- Sistemas basados en eventos para desacoplamiento extremo.
- Favorece resiliencia y observabilidad nativa con trazas distribuidas.
- Menos dependencia del acoplamiento síncrono.

### Edge Computing DevOps
- Despliegues distribuidos geográficamente cerca del usuario.
- Incluye automatización de CDN, lambdas en el edge y políticas de seguridad locales.

## Gestión del Conocimiento
### Runbooks y Playbooks Avanzados
- Runbooks automatizables (ej. scripts activados desde alertas).
- Playbooks para categorizar incidentes y aplicar patrones de respuesta iterables.

### Documentación Operativa Dinámica
- Integración de documentación continua con pipelines.
- Versionado automático de diagramas y flujos operativos.

## Cultura y Colaboración
### DevOps Maturity Models
- Evaluación por dimensiones: colaboración, automatización, calidad, entrega, observabilidad.
- Planes de mejora incrementales.

### Psychological Safety en DevOps
- Equipos que aceptan errores tempranos progresan más rápido.
- Correlación directa entre seguridad psicológica y velocidad de entrega.
- Base de los “blameless postmortems”.

### Flow Engineering
- Optimización del valor entregado: WIP limitado, análisis de cuellos de botella.
- Métricas de flujo (Flow Efficiency, Flow Load, Flow Time, Flow Velocity).

## Gobernanza y Compliance
### DevOps para Entornos Regulatorios
- Pipelines con auditoría, trazabilidad y segregación de funciones.
- Automatización de controles para PCI-DSS, ISO 27001, GDPR.

### Controles de Producción Automatizados
- Validación preventiva (pre-prod) basada en reglas de compliance.
- Firmas digitales para artefactos y cambios críticos.

## Observabilidad Avanzada
### Telemetría Inteligente
- Análisis de patrones utilizando ML para detectar anomalías y predecir fallos.
- Integración con trazas, logs y métricas con modelos probabilísticos.

### Tagging Estratégico
- Etiquetado consistente de recursos para correlación automática.
- Ideal para ambientes multi-cloud o multi-región.

## Fiabilidad y Escalabilidad
### Auto-remediación
- Scripts automáticos ante incidentes predecibles.
- Integración con alertas para cerrar el ciclo de sanación sin intervención.

### Escalado Predictivo
- Basado en patrones históricos y predicciones de carga.
- Evita saturación sin sobreaprovisionar.

## DataOps y MLOps (Extensiones Digitales)
### DataOps
- DevOps aplicado al ciclo de vida de datos.
- Incluye calidad, pipelines de ingesta, validación y despliegue de datasets.

### MLOps
- DevOps para modelos de machine learning.
- Versionado de datasets, automatización del entrenamiento, validación y despliegue.

## FinOps
- Prácticas para controlar y optimizar costes en la nube.
- Combina métricas de infra, presupuestos y decisiones basadas en uso real.
- Cultura compartida entre ingeniería y finanzas.

## Prácticas de Entorno Moderno
### Chaos Engineering
- Introduce fallos controlados para mejorar resiliencia.
- Validación del comportamiento ante degradaciones reales.

### Sustentabilidad y GreenOps
- Optimización de consumo energético en infraestructura.
- Análisis del coste ambiental del ciclo de vida de aplicaciones.

## Métricas Avanzadas de Performance del Equipo
- DORA Metrics avanzadas (variantes nuevas).
- SPACE Framework: satisfacción, rendimiento, actividad, comunicación, eficiencia.
- Enfoque holístico para medir salud del equipo DevOps.

# DevOps Tools

## Control de Versiones
- **Git**: sistema distribuido dominante para control de versiones.
- **GitHub / GitLab / Bitbucket**: plataformas de hosting, issues, merge requests y automatización.
- **Gitea / Forgejo**: alternativas autoalojadas ligeras.
- **SVN (legado)**: repos heredados aún presentes en sistemas antiguos.

## Automatización y Build
- **Jenkins**: servidor de automatización altamente extensible.
- **GitHub Actions**: automatización integrada en GitHub.
- **GitLab CI**: pipelines integrados nativamente.
- **Bitbucket Pipelines**: CI sencillo basado en YAML.
- **CircleCI / TravisCI**: plataformas CI basadas en cloud.
- **Bazel / Pants / Nx**: herramientas de build escalables para monorepos.
- **Maven / Gradle / npm / pnpm**: herramientas de build según lenguaje.

## Infraestructura como Código (IaC)
- **Terraform**: lenguaje declarativo multi-cloud.
- **Pulumi**: IaC con lenguajes generalistas (TypeScript, Python, Go).
- **Ansible**: configuración y automatización sin agentes.
- **Chef / Puppet**: herramientas clásicas de configuración.
- **SaltStack**: orquestación y automatización masiva.
- **CloudFormation / ARM / Deployment Manager**: IaC nativa de cada cloud.
- **Crossplane**: provisión declarativa integrada en Kubernetes.

## Contenedores y Orquestación
- **Docker / Podman / Buildah**: contenedores y creación de imágenes.
- **Kubernetes**: orquestación estándar del mercado.
- **K3s / K0s**: distribuciones ligeras de Kubernetes.
- **OpenShift**: plataforma empresarial basada en Kubernetes.
- **Nomad**: orquestador sencillo alternativo a K8s.
- **Harbor / GitHub Container Registry / ECR**: registries de imágenes.

## Service Mesh
- **Istio**: malla más madura y completa.
- **Linkerd**: ligera y enfocada en simplicidad.
- **Consul Connect**: mesh orientado a servicios HashiCorp.

## Observabilidad
### Métricas
- **Prometheus**: estándar para scraping de métricas.
- **VictoriaMetrics / Thanos / Cortex**: almacenamiento escalable de métricas.
- **StatsD / Telegraf**: recopiladores de métricas.

### Logs
- **ELK (Elasticsearch + Logstash + Kibana)**: pila de logs más conocida.
- **Loki + Promtail + Grafana**: logs económicos y escalables.
- **Fluentd / Fluent Bit**: agentes de ingestión de logs.

### Trazas
- **OpenTelemetry**: estándar de telemetría.
- **Jaeger / Tempo / Zipkin**: backend de trazas distribuidas.

### Dashboards
- **Grafana**: visualización universal.
- **Kibana**: analítica sobre Elasticsearch.

## Seguridad y DevSecOps
### Análisis de Código
- **SonarQube**: calidad y análisis estático.
- **Semgrep**: reglas SAST ligeras y rápidas.
- **CodeQL**: análisis avanzado de seguridad.

### Escaneo de Dependencias
- **Snyk**: vulnerabilidades de librerías.
- **Trivy**: escaneo de contenedores y dependencias.
- **Grype**: scanner CLI rápido para imágenes.

### Gestión de Secretos
- **Vault**: estándar empresarial.
- **SOPS**: cifrado de configuraciones GitOps.
- **Sealed Secrets**: gestión de secretos en Kubernetes.

### Seguridad en Kubernetes
- **Kyverno / OPA Gatekeeper**: políticas declarativas.
- **Falco**: detección de actividades sospechosas.

## Deployment y Release
- **ArgoCD / FluxCD**: herramientas GitOps.
- **Spinnaker**: despliegues progresivos multinube.
- **Octopus Deploy**: despliegues controlados para .NET y multiplataforma.
- **Helm**: empaquetado de aplicaciones K8s.
- **Kustomize**: configuración declarativa nativa en Kubernetes.

## Monitoreo de Infraestructura
- **Zabbix**: monitorización tradicional muy potente.
- **Nagios / Icinga / Centreon**: herramientas históricas de monitoreo activo.
- **Datadog / New Relic / Dynatrace**: plataformas APM comerciales integradas.
- **Netdata**: monitoreo en tiempo real de servidores.

## Gestión de Artefactos
- **JFrog Artifactory**: gestión universal de artefactos.
- **Nexus Repository**: repositorio de dependencias y contenedores.
- **GitHub Packages**: repos integrado con GitHub.

## Networking y Edge
- **Nginx / Apache**: servidores y reverse proxy.
- **Envoy**: proxy moderno para microservicios.
- **HAProxy**: balanceo extremadamente eficiente.
- **Cloudflare / Akamai / Fastly**: edge networks y seguridad.
- **Traefik**: reverse proxy nativo para K8s.

## Gestión de Configuración y Feature Flags
- **LaunchDarkly**: feature flags para progressive delivery.
- **Unleash**: alternativa open source.
- **Consul**: KV store + service discovery.
- **Etcd**: base de datos clave en Kubernetes.

## Bases de Datos y Pipelines de Datos
- **Flyway / Liquibase**: control de cambios en bases de datos.
- **Airflow / Prefect / Dagster**: orquestación de pipelines de datos.
- **Debezium**: CDC para integraciones event-driven.

## Mensajería y Middleware
- **Kafka**: streaming distribuido.
- **RabbitMQ**: mensajería tradicional robusta.
- **NATS**: mensajería ligera de alta velocidad.
- **Redis Streams**: soluciones para event-driven en memoria.

## Cost Management y FinOps
- **Kubecost**: análisis de costes en entornos Kubernetes.
- **CloudZero / CloudHealth**: optimización de costes en cloud.
- **Infracost**: coste estimado desde IaC.

## Herramientas de Pruebas
- **K6**: pruebas de carga modernas.
- **JMeter**: pruebas de rendimiento históricas.
- **Postman / Insomnia**: pruebas de API.
- **Cypress / Playwright**: pruebas e2e modernas.
- **TestContainers**: entornos efímeros para pruebas de integración.

## Productivity y Developer Experience
- **Tilt / Skaffold**: desarrollo ágil en Kubernetes.
- **Coder / GitPod**: entornos de desarrollo remotos reproducibles.
- **LocalStack**: emulación de AWS para testeo local.

## Gestión de Incidencias y On-Call
- **PagerDuty**: gestión de alertas y guardias.
- **OpsGenie**: on-call y escalados.
- **VictorOps / Splunk On-Call**: alternativas para incident response.
- **FireHydrant**: gestión del ciclo de vida de incidentes.

## Backups y Restauración
- **Velero**: backups de Kubernetes.
- **Restic**: backups ligeros.
- **BorgBackup**: almacenamiento cifrado deduplicado.


# DevOps Tools por Nivel de Madurez

## Nivel 1 — Fundamentos (Nacimiento DevOps)
Herramientas básicas para estandarizar código, automatizar tareas simples y crear visibilidad.

### Control de versiones
- Git (cualquier plataforma: GitHub / GitLab / Bitbucket)

### Automatización mínima
- GitHub Actions / GitLab CI / Bitbucket Pipelines (pipelines iniciales)
- Jenkins (cuando ya existe en la empresa)

### Build y dependencias
- npm / pnpm / Maven / Gradle  
- Docker (primeros contenedores)

### Monitoreo básico
- Netdata (servidor)
- Grafana (dashboards iniciales)

### Seguridad inicial
- Dependabot (GitHub)
- Snyk / Trivy (escaneos simples)

### Documentación
- Markdown + repositorio de documentación
- Diagramas con Mermaid

**Objetivo del nivel:** automatizar tareas manuales, estandarizar builds, introducir CI.

---

## Nivel 2 — Industrialización (DevOps Práctico)
Se introducen pipelines completos, IaC, logs centralizados y contenedores de forma estándar.

### CI/CD estable
- GitHub Actions / GitLab CI (pipelines multi-stage)
- Jenkins con pipelines declarativos

### Contenedores y Orquestación ligera
- Docker Compose
- Registries: GitHub Container Registry / GitLab Registry

### Infraestructura como Código
- Terraform (primeros módulos)
- Ansible (configuración y despliegues simples)

### Logs unificados
- ELK Stack (Elasticsearch + Logstash + Kibana)
- Loki + Promtail (alternativa ligera)

### Métricas
- Prometheus
- Grafana enriquecida

### Seguridad práctica
- SonarQube (calidad + SAST inicial)
- Trivy para contenedores

### Pruebas
- Postman
- JMeter o K6 para carga inicial

**Objetivo del nivel:** unificar despliegues, gestionar infra declarativa, consolidar observabilidad.

---

## Nivel 3 — Escalabilidad y Plataformas (DevOps Avanzado)
Se introducen Kubernetes, GitOps, trazas distribuidas y madurez en seguridad.

### Kubernetes como estándar
- Kubernetes (kubeadm, EKS, GKE, AKS)
- K3s / K0s (entornos ligeros)

### GitOps
- ArgoCD
- FluxCD

### Build y Artefactos
- Helm (paquetizado)
- Kustomize
- Artifactory / Nexus (repositorio universal)

### Observabilidad Completa
- OpenTelemetry (instrumentación)
- Jaeger / Tempo (trazas)
- Prometheus + Thanos / VictoriaMetrics (escalado)
- Grafana avanzado (alertas + correlación)

### Seguridad avanzada
- Vault (secretos)
- OPA Gatekeeper / Kyverno (políticas)
- Falco (runtime security)

### Service Mesh
- Linkerd (sencillo)
- Istio (completo)

### Pruebas y calidad
- Cypress / Playwright (e2e)
- TestContainers (pruebas de integración reproducibles)

**Objetivo del nivel:** estandarizar Kubernetes, automatizar despliegues declarativos, telemetría avanzada.

---

- Backstage (portal de servicios)
- Port (orquestación de recursos)
- Humanitec (plataforma IDP completa)

### Entornos efímeros
- Tilt / Skaffold (dev en Kubernetes)
- GitPod / Coder (entornos de desarrollo remotos)

### Configuración avanzada
- Consul (service discovery + KV)
- SOPS / Sealed Secrets (gestión declarativa de secretos)

### Gobernanza
- Crossplane (IaC dentro de Kubernetes)
- Policies as Code (OPA/Gatekeeper centralizado)

### Feature Flags
- LaunchDarkly
- Unleash (open source)

### Cost Management (FinOps)
- Kubecost
- CloudZero

**Objetivo del nivel:** autoservicio, estandarización organizacional, eficiencia de costes y governance.

---

## Nivel 5 — DevOps Autónomo (NoOps / Continuous Verification)
Nivel orientado a operaciones inteligentes, ML, automatización y experimentación segura.

### Observabilidad inteligente
- APM avanzado: Dynatrace / Datadog / New Relic
- Detección de anomalías con ML integrado

### Continuous Verification (CV)
- Keptn / Harness (verificación automática)
- Integración con SLOs para promoción automática

### Progressive Delivery
- Argo Rollouts (canary + blue/green inteligente)
- Flagger (canary con métricas)

### Auto-remediación
- Módulos con Terraform + eventos
- Alertas conectadas a automation runbooks

### Serverless & Edge
- Cloudflare Workers / Fastly Compute / AWS Lambda
- Automatización basada en eventos

### Chaos Engineering
- LitmusChaos
- Gremlin

**Objetivo del nivel:** minimizar intervención humana, decisiones basadas en datos, resiliencia continua.

---

## Resumen por Niveles
- **Nivel 1:** herramientas básicas → CI simple, documentación, build.  
- **Nivel 2:** IaC, logging unificado, contenedores productivos.  
- **Nivel 3:** Kubernetes, GitOps, telemetría avanzada.  
- **Nivel 4:** plataformas internas, autoservicio y gobernanza.  
- **Nivel 5:** automatización inteligente, progressive delivery, NoOps.

	- Automatización de tasks repetitivas.

	- Implementación de alertas y dashboards de desempeño.

	- Chaos Engineering: inyección de fallos controlados.
	- Auto-remediación mediante scripts y herramientas de orquestación.
- **Incidentes y post-mortems**
	- Ejecución de blameless postmortems tras incidentes.
	- Análisis de métricas para prevenir futuros errores.
- **Escalabilidad**
	- Planificación de auto-scaling y balanceo de carga.
	- Gestión de entornos multi-región para alta disponibilidad.

## DevSecOps Engineer
- **Integración de seguridad en pipelines**
	- Escaneos SAST, DAST y SCA automatizados.
	- Configuración de gates de seguridad antes de deploy.
- **Gestión de secretos**
	- Uso de [DevSecOps](/devops/devsecops/) y herramientas como Vault, SOPS.
	- Control de acceso a credenciales y certificados TLS.

	- Automatización de revisiones de cumplimiento (PCI-DSS, GDPR).

	- Uso de OpenTelemetry, Jaeger o Tempo para correlación de eventos.

- **Performance tuning**
	- Identificación de cuellos de botella.
	- Recomendación de mejoras de infraestructura o código.

## Release / Deployment Engineer

## HTTP
El protocolo HTTP es la base de la comunicación web. Define cómo los clientes y servidores intercambian solicitudes y respuestas usando métodos estandarizados, encabezados y un modelo stateless.
### Conceptos
Get / HTTP/1.1  
Host: 74.125.224.72  
client → modem → ISP DNS → authoritative DNS  
client → modem → server → client
### Detalles ampliados
- Métodos comunes  
- GET, POST, PUT, DELETE, PATCH, OPTIONS, HEAD
- Versiones  
- HTTP/1.1, HTTP/2, HTTP/3
- DNS  
- A, AAAA, CNAME, MX, TXT, SRV, NS

## HTTPS
HTTPS añade cifrado y autenticación sobre HTTP mediante SSL/TLS. Garantiza confidencialidad, integridad y autenticidad de las comunicaciones.
### Conceptos
SSL

TLS

## SMTP
SMTP es el protocolo usado para enviar correos electrónicos. Se complementa con POP3 e IMAP para la recepción.
### Flujo local
sender → SMTP Server → POP3/IMAP → recipient
### Flujo outbound
sender → SMTP Server → ISP DNS lookup → 2nd SMTP Server → POP3/IMAP → recipient
### Detalles ampliados
- POP3 vs IMAP  
- SPF / DKIM / DMARC  
- MTAs (Postfix, Exim)

Una LAN es una red local usada para conectar dispositivos dentro de un hogar, oficina o entorno cerrado, normalmente con direccionamiento privado.
### Conceptos
ISP → modem → coaxial/DSL → modem → ethernet  
router (IP local)  
IP pública del ISP
### Detalles ampliados
- Clases de red  
- DHCP  
- Switches L2/L3, VLANs, STP
